

### 1. Ядро системы: Backend & AI

Проект завязан на компьютерное зрение, генерацию контента и обработку данных в реальном времени. В этой сфере безоговорочно лидирует экосистема Python.

* **Python 3.11 + FastAPI:** Для обработки телеметрии от робота и статусов прогулок нам нужен быстрый асинхронный сервер. FastAPI идеально для этого подходит — он работает быстро и легко масштабируется. Версия 3.11 дает отличный прирост в скорости работы по сравнению со старыми версиями.
* **Pydantic:** В системе много критичных данных (профили собак с их здоровьем, координаты, статусы заряда). Pydantic обеспечит строгую валидацию этих данных, чтобы неверный тип переменной не «положил» сервер во время прогулки.
* **PostgreSQL:** Самая надежная реляционная СУБД с открытым исходным кодом. Для нас критически важно наличие расширения *PostGIS*, которое превращает базу в мощный гео-движок. Это закроет все требования по хранению и перестроению маршрутов (ФТ 6.1.2).
* **YOLOv8 (Object Detection):** Требование НФТ 6.2.2 диктует реакцию на угрозы быстрее 1 секунды. Модели семейства YOLO (You Only Look Once) — это индустриальный стандарт для распознавания объектов (собак, машин, людей) в реальном времени. Восьмая версия легкая и отлично справится с задачей.
* **OpenAI API:** Для автоматического формирования текстовых отчетов по итогам прогулки (ФТ 6.1.4) нет смысла обучать свою языковую модель. Дешевле и надежнее использовать готовое API от OpenAI, передавая туда сухие логи с датчиков для генерации красивого текста.

### 2. Пользовательский интерфейс: Frontend

Для быстрого тестирования гипотез и запуска прототипа логичнее начать с кроссплатформенного веб-интерфейса, который можно будет обернуть в приложение.

* **HTML5, CSS3, JavaScript (ES6):** Золотой стандарт веба. Современный JS позволяет делать быстрые и отзывчивые интерфейсы. Главное — писать чистый код без лишних комментариев и использовать безопасные методы работы с DOM (например, избегать использования `innerHTML` для рендеринга данных о собаках, чтобы не словить уязвимости).
* **TailwindCSS:** Написание кастомного CSS с нуля займет слишком много времени. Tailwind позволит быстро собрать адаптивный интерфейс с помощью utility-классов прямо в HTML.
* **OpenStreetMaps API:** Идеальное решение для построения маршрутов выгула. В отличие от коммерческих аналогов, это бесплатно, гибко и позволяет накладывать свои слои (например, опасные зоны или плотность движения).
* **Хостинг (GitHub Pages / Vercel):** Требование НФТ 6.2.1 просит доступность не ниже 99%. Размещение собранного фронтенда на Vercel или GitHub Pages закрывает эту задачу бесплатно, предоставляя глобальный CDN и автоматический деплой при обновлении кода.

### 3. Инструменты и управление (Организация работы)

Чтобы команда не тратила время на настройку процессов, берем проверенную классику:

* **Git / GitHub:** Очевидный стандарт для контроля версий и совместной работы.
* **GitHub Projects (Канбан-доска):** Встроенные проекты в GitHub позволяют держать карточки задач прямо рядом с кодом и коммитами.
* **Telegram:** Отлично подходит для рабочего чата и мгновенных уведомлений. Более того, через Telegram-ботов можно реализовать часть интерфейса для экстренного управления или оповещения владельцев об угрозах.
* **Discord:** Идеален для Daily-митингов и созвонов. Качественная связь, удобные голосовые «комнаты» и возможность быстро расшарить экран с кодом делают его удобнее классического Zoom.
